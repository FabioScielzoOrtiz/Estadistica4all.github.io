Aplicación del aprendizaje automático en la medicina predictiva.¶

Práctica 1: Clasificación para el diagnóstico de enfermedades.¶

Agenda¶

  1. Objetivos
  2. Problema
  3. Datos
  4. Desarrollo de la prctica

1. Objetivos.¶

  • Conocer un contexto de aplicacin real.

  • Ejercitarnos en el anlisis de datos y la implementacin de algoritmos de clasificacin, para adquirir criterio en la aplicacin de los mismos.

  • Evaluar las capacidades de R y mlr para la implementacin.

2. Problema.¶

  • Resolver un problema de clasificacin para el diagnstico de pacientes hepticos.

3. Datos.¶

  • Usaremos el conjunto de datos Indian Liver Patient Dataset: Los pacientes con enfermedades del hgado han ido aumentando continuamente debido al consumo excesivo de alcohol, inhalacin de gases nocivos, ingesta de alimentos contaminados, encurtidos y drogas.

  • Este conjunto de datos se utiliz para evaluar los algoritmos de prediccin en un esfuerzo por reducir la cargapara los mdicos. Este conjunto de datos contiene 416 registros de pacientes hepticos y 167 registros de pacientes no hepticos recopilados en el noreste de Andhra Pradesh, India.

  • La variable de respuesta es diseased (personas que tienen enfermedad del hgado)

  • El data set encuentra en la librera mlr3data.

data("ilpd", package = "mlr3data")

Nos apoyaremos en la funcion cut() de la libreria Pandas para categorizar la variable Age usando la regla de los cuartiles.

A esta funcion le das un vector (bins) y construye unos intervalos con los elementos del vector, en este caso (3, 33], (33, 45], (45, 58], (58, 90]. arbol_0<-rpart(diseased~.,data = datos_entreno, method = "class",cp=0.01)
rpart.plot(arbol_0, 
           extra = 104,          # show fitted class, probs, percentages
           box.palette = "GnBu", # color scheme
           branch.lty = 3,       # dotted branch lines
           shadow.col = "gray",  # shadows under the node boxes
           nn = TRUE)

(prediccion1<-predict(arbol_0,newdata=datos_test,type="class")) # Predicciones y matriz de confusin
(matriz_confusion1<-confusionMatrix(prediccion1,datos_test[["diseased"]]))


# Debido a la alta complejidad de estos arboles le vamos a hacer un proceso de prepoda, para ello podemos hacer uso tanto del parmetro cp o directamente manipulando los hiperparametros de los modelos.


# Arbol un poco podado (no se pide en la prctica)

set.seed(1)

datos_test2<-setdiff(ilpd, datos_entreno2) # Separamos los datos de test

arbol_1<-rpart(diseased~.,data=datos_entreno2,maxdepth=4,method = "class") # Cambiamos la profundidad
rpart.plot(arbol_1, 
           extra = 104,          # show fitted class, probs, percentages
           box.palette = "GnBu", # color scheme
           branch.lty = 3,       # dotted branch lines
           shadow.col = "gray",  # shadows under the node boxes
           nn = TRUE)

# Con esto hemos conseguido un modelo mucho ms simple que el anterior sin prepoda.

(prediccion2<-predict(arbol_1,newdata=datos_test2,type="class")) # prediccin y matriz de confusin
(matriz_confusion2<-confusionMatrix(prediccion2,datos_test2[["diseased"]]))

Se puede ver como hemos mejorado un poco la precisin de la prediccin simplemente cambiando el coeficiente de particin. Cabe remarcar que el algoritmo de los CART utilizan Gini como criterio de divisin.
